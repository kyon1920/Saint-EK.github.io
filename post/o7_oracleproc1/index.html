          proc的数据类型被分为 内部数据类型、外部数据类型、宿主变量
          oracle 内部不会对数据类型进行转化，全靠 proc 编译器转换实现

          BLOB：大批量二进制数据，最大 4G
          BFILE：OS文件数据
          NCHAR，NVARCHAR2,NCLOB：本地字符集数据

    proc 数据类型转换
    👸 我们程序员想要对 proc 数据类型进行转换，只能从 宿主变量 → 外部数据类型 进行干预和转换，其余的都由Oracle内部机制完成，程序员无法干预。
          定义宿主变量：定义的变量，既可以在C/C++中使用，也可以在嵌入式SQL语言中使用
    👸 eg：数组数据类型
            typedef char dnameType[20];
            typedef char locType[20]; // 声明数组，数据类型(本质：固定大小内存块的别名)
         由以下语句可以将 char[] 转换成为 string 数据类型
            EXEC SQL BEGIN DECLARE SECTION;
            char * ssid = "oubc/oubc";

            // 编译器自动转换
            varchar dname_1[20];

    👸 什么叫可以转换，可以将上表中 宿主变量与Oracle的外部数据类型 的类型进行相互转换
    👸 利用 select 获取 char[] 类型的值会在数组不完全部分填充空格，而 varchar[] 和 string 类型则不会出现这样的情况

    指示变量的作用
         指示变量即是，如果想往数据库中写入一个空值，必须使用指示变量告诉编译器这个值是一个空值
         如果在读数据库的时候，读到的值是空值，编译器会将指示变量赋值为 -1
         语法：
             short loc_ind;     定义指示变量只能使用 short 类型
    👸 eg：
         EXEC SQL BEGIN DECLARE SECTION;
         int loc[20];
         short loc_imd; // 用short类型定义了一个loc变量的指示变量，用这个变量来指示这个变量是空还是非空，当loc做输入时且loc_imd为-1时表示插入一个空值
             EXEC SQL END DECLARE SECTION;
             . . . . . .
         使用：
         插入：EXEC SQL insert into student values(:ID, :name, :loc:locimd)
         获取：
             EXEC SQL select id, name, age into getid, :getname:loc_imd, getage from student where id='XXX';
             if (loc_imd == -1)
             {
                   strcpy(getname, "null");
             }

    proc的通讯区和错误处理机制
    👸 上面的程序中，使用了 sqlca.sqlcode，则是使用了错误处理
    👸 当在 Pro * C/C++ 应用程序中运行SQL语句时，Oracle会将最近执行的SQL语句的状态信息存储到变量SQLCODE、SQLSTATE或SQLCA 结构中，当SQL语句执行成功时，一般情况下不需要进行其它处理，当SQL执行失败时，应用程序能够检测到失败信息，并进行适当的处理，以加强应用程序的健壮性。在编写 Pro * C/C++ 程序时，通过使用 WHENEVER 语句可以检测并处理SQL错误，语法如下：
           其中，condition 用于指定要检测的条件，action 用于指定满足特定条件时要执行的操作。
           1>. 条件 (condition)
                当编写数据库应用程序时，可能出现的SQL异常包括警告、严重错误和未检索到数据 三种情况，在编写Pro * C/C++应用程序时，可以成功地检测到这些异常情况，异常条件如下：
                • SQLWARNING：用于检测Oracle警告，当检测到该条件时，sqlca.sqlwarn(0)会被设置为W，注意，当使用该条件时，必须定义SQLCA结构；
                • SQLERROR：用于检测Oracle错误，当检测到该条件时，sqlca.sqlcode 或 SQLCODE 会被设置为负值
                • NOT FOUND：用于检测 SELECT 或 FETCH 操作未检索到行的情况，当MODE=ORACLE时，如果不能找到满足条件的行，则 sqlca.sqlcode 会被设置为 +1403，当MODE=ANSI时，SQLCODE会被设置为+100；
           2>.操作 (action)
                当Oracle检测到SQL异常情况时，应用程序应该编写特定操作来处理异常情况，当编写Pro * C/C++应用程序时，检测到异常之后可以执行以下操作：
                • CONTINUE：继续运行下一条语句；
                • DO：执行错误处理函数
                • GOTO label_name：转移到标号处执行
                • STOP：回退事务，停止执行
           3>.WHENEVER 语句范围
                WHENEVER 是定义语句，它会自动检测应用程序中的所有SQL语句，当编写Pro * C/C++ 应用程序时，WHENEVER 应该放在第一条内嵌SQL语句(eg：CONNECT)之前，如果在应用程序中使用了多条 WHENEVER 语句，则在进行随后的处理时，只有最近的 WHENEVER 语句起作用
    👸 做一个错误处理函数，这样就不必要每次都使用 sqlca.sqlcode 进行判断
           EXEC SQL WHENEVER SQLERROR(条件) DO(动作) sqlerr();
           即当出现 SQLERROR 条件时则执行 sqlerr() 函数
           // 打印出错原因

    {
                /*下一步，如果此函数中的EXEC SQL又出错了，
                就会形成间接递归而形成死循环，
            故这句话的作用范围在下方的EXEC SQL语句，

    普通游标
    ﹢ 如果数据量太大，我们不可能定义一个上万行的数组来对数据库进行操作，游标就是为了解决这样的问题
    ﹢ 游标即是一行一行的获取数据
    ﹢ 游标是为某次查询而生
    👸 游标操作数据步骤：
    1 定义游标
          EXEC SQL DECLARE emp_cursor CURSOR FOR;
          select id, name, age from student;
    2 打开游标
          EXEC SQL OPEN emp_cursor;
          (Open cursor: put the select results into a memory place and the cursor pointer points to the first row data.)
    3 提取数据
          while(1)
          EXEC SQL FETCH emp_cursor INTO :empid, :empname, :empage;
          /* 处理数据类型 */
          (After fetch, cursor pointer moves down one line. Use loop for multiple rows)
          // 每一次处理游标都将移向下一行
          // 怎么判断游标已经到了最后一行数据呢？利用 if (sqlca.sqlcode == 100 || sqlca.sqlcode == 1403)
          // 两个错误码 是由于Oracle版本的历史原因形成的，ANSI组织规定为100，Oracle厂商规定为1403
          break;
    4 关闭游标
          EXEC SQL CLOSE emp_cursor

    滚动游标
    ﹢ 可以滚动到指定的任意一行
    👸 游标操作数据步骤：
    1 定义游标
          EXEC SQL DECLARE emp_cursor SCROLL CURSOR FOR;
          select id, name, age from student;
    2 打开游标
          EXEC SQL OPEN emp_cursor;
          (Open cursor: put the select results into a memory place and the cursor pointer points to the first row data.)
    3 提取数据
          while(1)
          // 查询下一行
          EXEC SQL FETCH NEXT emp_cursor INTO :empid, :empname, :empage;
          // 查询上一行
          EXEC SQL FETCH PRIOR emp_cursor INTO :empid, :empname, :empage;
          // 查询第一行
          EXEC SQL FETCH FIRST emp_cursor INTO :empid, :empname, :empage;
          // 查询最后一行
          EXEC SQL FETCH LAST emp_cursor INTO :empid, :empname, :empage;
          // 查询绝对值第三行，即第三行
          EXEC SQL FETCH RELATIVE 3 emp_cursor INTO :empid, :empname, :empage;
          // 查询相对第三行，即当前行往后数第三行
          EXEC SQL FETCH NEXT emp_cursor INTO :empid, :empname, :empage;
          /* 处理数据类型 */
          (After fetch, cursor pointer moves down one line. Use loop for multiple rows)
          // 每一次处理游标都将移向下一行
          // 怎么判断游标已经到了最后一行数据呢？利用 if (sqlca.sqlcode == 100 || sqlca.sqlcode == 1403)
          // 两个错误码 是由于Oracle版本的历史原因形成的，ANSI组织规定为100，Oracle厂商规定为1403
          break;
    4 关闭游标
          EXEC SQL CLOSE emp_cursor

    Oracle存储中文乱码问题
    👸 乱码出现的可能：远程连接工具(windows下的)编码与Oracle服务器不一致，sqlplus工具，写数据到Oracle数据库，Oracle服务器与数据库服务器之间
          在项目开发工程中只能一个一个进行排查
    👸 解决思路一：利用 sqlplus 工具插入中文数据，查看是否出现乱码，如果出现了乱码就是这里的问题，可以先不用管其它地方
          sqlplus 通过 .bash_profile 中的 NLS_LANG 环境决定显示语言
          NLS_LANG="SIMPLIFIED CHINESE_CHINA.ZHS16GBK"
          export NLS_LANG
          代表简体中文 gbk
          BLS_LANG="SIMPLIFIED CHINESE_CHINA.AL32UTF8"
          代表简体中文 utf8
          系统中文：export LANG="zh_CN.UTF8"
    👸 解决思路二：windows 连接工具 / 应用程序编辑器 保存文件到服务器出现中文乱码
          通过应用程序执行插入语句观测结果，是否出现乱码
          出现乱码，即C/C++源码在windows上转换后即是服务器上的乱码
          解决：把windows上的编辑器编码方式设置为：( ASCII 转 UTF-8 ) Unicode 编辑
