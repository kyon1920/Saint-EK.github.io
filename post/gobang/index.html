红蓝五子棋

2019.01.24

经典小游戏：童年的五子棋



项目名称：红蓝五子棋
开发工具：VS2019
开发语言：C/C++、windowsAPI
项目介绍：游戏分为两方，一为红方，一为蓝方，任意一方先下棋，另一方继续下，直到某一方率先筹齐五子连珠者胜。

具体实现：
1、游戏棋盘：15X15正方形网格图形，由windowsAPI-Rectangle()函数实现最外层框图，由windowsAPI-MoveToEx()和LineTo()两个函数结合循环画线实现内部正方形网格现，该实现被放在PAINT消息里面。

2、实现逻辑：定义一个15x15的二维数组，并将他们全部初始化为0，由鼠标左键实现在棋盘中放置红棋，当玩家在相应位置放置红棋时，对应该数组元素设置为1，由鼠标右键实现在棋盘中放置蓝棋，当玩家在相应位置放置蓝棋时，对应该数组元素设置为2，每次下棋后(无论是红棋还是蓝棋)，都要进行该处是否被对方棋子占据和五子连珠检查，该棋盘格子处未被占据并且未形成五子连珠方可继续在该处放置棋子，当该位置被对方棋子占据时，另选位置放置棋子即可，若是形成了五子连珠，胜负决出，游戏结束。

3、实现分析：
    a)待优化处：应将棋盘的实现、判断五子连珠单独写在一个函数中，这样可以避免代码重复，减少工作量。
    b)优点：每次利用鼠标点击来放置棋子时，并不是鼠标点哪里就在哪里放置棋子，而是首先对鼠标点击位置进行判断，只有当鼠标点击位置在棋盘格子的左右上下15个像素点的时候，才会在棋盘放置以该格子为中心，半径为15的圆形相应颜色的棋子，从而避免了鼠标点哪里就在哪里放置棋子。

// “关于”框的消息处理程序。
INT_PTR CALLBACK About(HWND hDlg, UINT message, WPARAM wParam, LPARAM lParam)
{
    UNREFERENCED_PARAMETER(lParam);
    switch (message)
    {
    case WM_INITDIALOG:
        return (INT_PTR)TRUE;

    case WM_COMMAND:
        if (LOWORD(wParam) == IDOK || LOWORD(wParam) == IDCANCEL)
        {
            EndDialog(hDlg, LOWORD(wParam));
            return (INT_PTR)TRUE;
        }
        break;
    }
    return (INT_PTR)FALSE;
}
