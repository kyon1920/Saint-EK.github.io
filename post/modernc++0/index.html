0、介绍
正则表达式(Regular Expression)描述了一种字符串匹配的模式(Pattern)，可以用来检查一个串是否含有某种子串、将匹配的子串或者从某个串中取出符合某个条件的子串等。
正则表达式十分强大，具体的实现算法有一定差异，C++新标准支持6中正则表达式引擎，ECMAScript是其中支持最多元的引擎，也是Regex默认支持的引擎。
     ● ECMAScript
     ● basic (POSIX Basic Regular Expressions)
     ● extended (POSIX Extended Regular Expressions)
     ● awk (POSIX awk)
     ● grep (POSIX grep)
     ● egrep (POSIX grep –E)

1、正则表达式语法
正则表达式主要由两部分构成，特殊字符和普通字符

符号 意义 描述
        .   匹配除"\n"之外的任何单个字符 "[\s\S]"之类的模式可以匹配包括"\n"在内的任意字符
        ^   匹配输入字符串的开始位置 当该符号在方括号表达式中使用时，表示不接受该方括号表达式中的字符集合。要匹配"^"字符本身，请使用"\^"
        $   匹配输入字符串结尾的位置 要匹配"$"字符本身，请使用"\ \$"
        *   零次或多次匹配前面的字符或子表达式 等价于"{0,}"要匹配"*"字符本身，请使用"\*"
        +   一次或多次匹配前面的字符或子表达式 等价于"{1,}"要匹配"+"字符本身，请使用"\+"
        ?   零次或一次匹配前面的字符或子表达式 等价于"{0,1}"，要匹配"?"字符本身，请使用"\?"
        |   将两个匹配条件进行逻辑"或"运算 指明两项之间的一个选择，要匹配"|"字符本身，请使用"\|"
        \   将下一字符标记为特殊字符、文本、反向引用或八进制转义符 如： "n" 匹配字符 "n"，"\n" 匹配换行符，序列 "\\" 匹配 "\"，而 "\(" 则匹配 "("
      \w    匹配任何一个字母或数字或下划线 即 A~Z，a~z，0~9，_ 中任意一个
      \W    匹配任意不是字母、数字、下划线的字符
      \s    匹配任意的空白符 等价于"[\f\n\r\t\v]"，包括空格、制表符、换页符等
      \S    匹配任意不是空白的字符 等价于"[^\f\n\r\t\v]"
      \d    匹配数字 等价于"[0-9]"，0-9中任意一个数字
      \D    匹配任意非数字的字符 等价于"[^0-9]"
      \b    匹配一个字边界，即字与空格间的位置 也就是单词和空格之间的位置，如："er\b"匹配"never"中的"er"，但不匹配"verb"中的"er"
      \B    非字边界匹配 "er\B"匹配"verb"中的"er"，但不匹配"never"中的"er"
      \f    匹配一个换页符 等价于"\x0c"和"\cL"
      \n    匹配一个换行符 等价于"\x0a"和"\cJ"
      \r    匹配一个回车符 等价于"\x0d"和"\cM"
      \t    匹配一个制表符 等价于"\x09"和"\cl"
      \v    匹配一个垂直制表符 等价于"\x0b"和"\cK"
     \cx    匹配"x"指示的控制字符 如：\cM匹配Control-M或回车符，"x"的值必须在"A-Z"或"a-z"之间,如果不是这样，则假定c就是"c"字符本身
     {n}    "n"是非负整数，正好匹配n次 如："o{2}"与"Bob"中的"o"不匹配，但与"food"中的两个"o"匹配
     {n,}    "n"是非负整数，至少匹配n次 "o{1,}"等价于"o+"，"o{0,}"等价于"o*"，如："o{2,}"不匹配"Bob"中的"o"，而匹配"foooood"中的所有"O"
    {n,m}    "n"和"m"都是非负整数n<=m，匹配至少n次，至多m次 如："o{1,3}"匹配"fooood"中的头三个o，"o{0,1}"等效于"o?"，注意，不能将空格插入逗号和数字之间
      x|y    匹配"x"或"y" 如："z|food"匹配"z"或"food"，"(z|f)ood"匹配"zood"或"food"
     [xyz]   字符集，匹配包含的任一字符 如："[abc]"匹配"plain"中的"a"
    [^xyz]   反向字符集，匹配未包含的任意字符 如："[^abc]"匹配"plain"中的"p"
     [a-z]   字符范围，匹配指定范围内的任何字符 如："[a-z]"匹配"a"到"z"范围内的任何小写字母
    [^a-z]   反向范围字符，匹配不在指定的范围内的任何字符 如："[^a-z]"匹配任何不在"a"到"z"范围内的任何字符
       ()  将"("和")"之间的表达式定义为"组"group 匹配这个表达式的字符保存到一个临时区域，一个正则表达式中最多能保存9个，可以用"\1"到"\9"的符号来引用
 (pattern)  匹配pattern并捕获该匹配的子表达式 可以使用\$0…\$9属性从结果"匹配"集合中检索捕获的匹配；"\$0"表示整个匹配，"\$1"表示第1个子表达式的匹配结果，当只有一个子表达式时两者可能是相同
(?:pattern) 匹配pattern但不捕获该匹配的子表达式 即它是一个非捕获匹配，不存储供以后使用的匹配，这对于用"or"字符 "(|)"组合模式部件的情况很有用, 如："industr(?:y|ies)"是比"industry|industries"更简略的表达式
(?=pattern) 非获取匹配，正向肯定预查，在任何匹配pattern的字符串开始处匹配查找字符串，该匹配不需要获取供以后使用 如："Windows(?=95|98|NT|2000)"能匹配"Windows2000"中的"Windows"，但不能匹配"Windows3.1"中的"Windows”。预查不消耗字符，也就是说，在一个匹配发生后，在最后一次匹配之后立即开始下一次匹配的搜索，而不是从包含预查的字符之后开始
(?!pattern) 非获取匹配，正向否定预查，...... 同上 如："Windows(?!95|98|NT|2000)"能匹配"Windows3.1"中的"Windows"，但不能匹配"Windows2000"中的"Windows"

注："?" 匹配模式是"非贪心的"。"非贪心的" 模式匹配搜索到的、尽可能短的字符串，而默认的"贪心的"模式匹配搜索到的、尽可能长的字符串。如，在字符串"oooo"中，"o+?“只匹配单个"o”，而"o+“匹配所有"o”；

2、正则表达式的使用
● 目标序列(Target sequence)：为模式搜索到的字符序列，可以是2个迭代器所指定的范围、空终止字符串或一个 std::string 。
● 模式(Pattern)：即正则表达式，是带特定语法的字符串构成的 std::basic_regex 类型对象，支持的语法变体描述见 syntax_option_type 。
● 匹配的数组(Matched array)：关于匹配的信息可作为 std::match_results 类型对象获取。
● 替换字符串(Replacement string)：确定如何替换匹配的字符串，受支持的语法变体的描述不同而不同。

4、算法
封装于regex的正则表达式应用到字符的目标序列，算法主要由函数模板实现
● regex_match：试图匹配正则表达式到整个字符序列
● regex_search：试图匹配正则表达式到字符序列的任何部分
● regex_replace：以格式化的替换文本来替换正则表达式匹配的出现位置

5、迭代器
迭代器用于遍历在序列中找到的匹配正则表达式的整个集合
● regex_iterator：在字符序列中通过所有正则表达式匹配迭代
● regex_token_iterator：通过在给定的字符串中所有正则表达式匹配中的指定子表达式，或通过不匹配的子串迭代

6、异常
● regex_error：定义一个对象抛出的来自正则表达式库的异常

7、分组及其它概念
● 分组：用小括号来指定子表达式的重复次数，如：IP地址匹配表达式 (\d{1,3}.){3}\d{1,3}
● 后向引用：使用小括号指定一个子表达式后，匹配这个子表达式的文本可以在表达式或其它程序中作进一步的处理，默认情况下，每个分组会自动拥有一个组号，规则是：以分组的左括号为标志，从左向右，第一个分组的组号为1，第二个为2，以此类推。后向引用用于重复搜索前面某个分组匹配的文本，如：\1代表分组1匹配的文本
● 分组及捕获：
表达式 意义
(exp) 匹配exp，并捕获文本到自动命名的组里
(?exp) 匹配exp，并捕获文本到名称为name的组里
(?:exp) 匹配exp，不捕获匹配的文本
(?=exp) 匹配exp前面的位置
(?<=exp) 匹配exp后面的位置
(?!exp) 匹配后面跟的不是exp的位置
(?<!exp) 匹配前面不是exp的位置
(?#comment) 这种类型的组不对正则表达式的处理产生任何影响，只是为了提供让人阅读注释
● 贪婪与懒惰 贪婪匹配：当正则表达式中包含能接受重复的量词(指定数量的代码，例如*,{5,12}等)时，通常的行为是匹配尽可能多的字符。考虑这个表达式：a.*b，它将会匹配最长的以a开始，以b结束的字符串。如果用它来搜索aabab的话，它会匹配整个字符串aabab。 懒惰匹配：匹配尽可能少的字符。前面给出的量词都可以被转化为懒惰匹配模式，只要在它后面加上一个问号?。这样.?就意味着匹配任意数量的重复，但是在能使整个匹配成功的前提下使用最少的重复。如：a.?b匹配最短的，以a开始，以b结束的字符串。如果把它应用于aabab的话，它会匹配aab和ab。
表达式 意义
? 重复任意次，但尽可能少重复
+? 重复1次或更多次，但尽可能少重复
?? 重复0次或1次，但尽可能少重复
{n,m}? 重复n到m次，但尽可能少重复
{n,}? 重复n次以上，但尽可能少重复

    cout << boolalpha << regex_match(str, re0) << endl;

    return 0;
}

#include <iostream>
#include <regex>
#include <string>
using namespace std;

auto main() -> int
{
    // 匹配所有的小写字母，替换为*
    string str{ R"(\abcde Hello world! _123456\m\n)" };
    string str_replace{ "*" };
    regex rep(R"([a-z])");
    cout << regex_replace(str, rep, str_replace) << endl;
    // 输出 \***** H**** *****! _123456\*\*

    // \$&表示匹配的字符,[\$&]即替换为[匹配的字符]
    str_replace = "[$&]";
    cout << regex_replace(str, rep, str_replace) << endl;
    // 输出只要有字母的地方加上[]括号括起来

    // $1 $2 $3$4表示正则表达式中对应的4个捕获组 (子表达式)
    // 匹配之后按要求显示4个捕获组内容
    str = "{0xff-0x11-0xab-0x21}";
    str_replace = "$1 $2 $3$4";
    rep = "\\{([a-z0-9]+)-([a-z0-9]+)-([a-z0-9]+)-([a-z0-9]+)\\}";
    // () 括起来的为子表达式，对应着对应的$n 
    cout << regex_replace(str, rep, str_replace) << endl;
    // 输出 0xff 0x11 0xab0x21
    str_replace = "$1 $3$4";
    rep = "\\{([a-z0-9]+)-([a-z0-9]+)-([a-z0-9]+)-([a-z0-9]+)\\}";
    cout << regex_replace(str, rep, str_replace) << endl;
    // 输出 0xff 0xab0x21

    str = "{0xff-0x11-0xab-0x21}";
    rep = "([a-z0-9]+)-([a-z0-9]+)-([a-z0-9]+)-([a-z0-9]+)";
    str_replace = "$(反撇号)";  // 将匹配的内容替换成prefix
    string str_replace1 = "$'";  // 将匹配的内容替换成suffix
    // 参数3为"$(反撇号)"，与正则表达式相匹配的子序列之前的字符序列
    cout << regex_replace(str, rep, str_replace) << endl;
    // 输出 {{}
    // // 参数3为"$'"，与正则表达式相匹配的子序列之后的字符序列
    cout << regex_replace(str, rep, str_replace1) << endl;
    // 输出 {}}

    // 修改默认的flag参数为format_no_copy,即不拷贝prefix和suffix
    cout << regex_replace(str, rep, str_replace, 
                        regex_constants::format_no_copy) << endl;
    // 输出 {
    cout << regex_replace(str, rep, str_replace1, 
                        regex_constants::format_no_copy) << endl;
    // 输出 }

    return 0;
}
ECMAScript 扩展格式规则 替换的字符
$& 与整个正则表达式相匹配的字符序列
$$ 直接替换$
$`(反撇号) 与正则表达式相匹配的子序列之前的字符序列
$’(正撇号) 与正则表达式相匹配的子序列之后的字符序列
$n 在 n(即0-9)位置与捕获组相匹配的字符序列
$nn 在 nn(即10-99)位置与捕获组相匹配的字符序列

#include <iostream>
#include <regex>
#include <string>
using namespace std;

auto main() -> int
{
    string str = "Quick brown fox.";
    regex rep = "[^\\s]+";
    // 遍历整个字符串，匹配到空白时进入下一次匹配
    // 匹配的结果存放在迭代器中
    auto wordIt = sregex_iterator(str.begin(), str.end(), rep);

    // 计算迭代器中存放的正确匹配的数量
    cout << distance(wordIt, sregex_iterator()) <<
            " words." << endl;
    // 输出 3 words
    // 进行遍历输出
    for (; wordIt != sregex_iterator(); ++wordIt)
    {
        cout << wordIt->str() << endl;
    }
    // 输出 Quick\nbrown\nfox.

    str = "AB-12-AB AB-23-AC";
    // (\\D{2})表示2个非数字,(\\d{2})表示2个数字，
    // (\\1)表示第1个捕获组即(\\D{2})的内容
    rep = "(\\D{2})-(\\d{2})-(\\1)";
    regex_iterator<string::const_iterator>
        begin(str.begin(), str.end(), rep);
    for (auto iter = begin; iter != sregex_iterator(); iter++)
    {
        cout << iter->str() << endl;
    }
    // 输出 AB-12-AB

    return 0;
}

#include <iostream>
#include <regex>
#include <string>
using namespace std;

auto main() -> int
{
    string str = "Quick brown fox.";
    string data{ "<person>\n"
    " <first>Nico</first>\n"
    " <last>Josuttis</last>\n"
    "</person>\n" };
    // \\1指定第1个捕获组,即第1个子表达式
    regex re("<(.\)>(.\*)</(\\1)>");

    // 0是默认值，即完整的匹配结果，
    // 1是匹配结果的第1个捕获组，
    // 2是匹配结果的第2个捕获组(子表达式)
    // iterate over all matches (using a regex_token_iterator):
    sregex_token_iterator pos(
        data.cbegin(), data.cend(),    // sequence
        re,     // token separator
        0);     // 0:full match, 
                //1:First substring, 2:second substring
    sregex_token_iterator end;
    for (; pos != end; ++pos)
    {
        cout << pos->str() << endl;
    }
    // 输出
    // <first>Nico</first>
    // <last>Josuttis</last>

    // 第4个参数支持数组
    int arrSubMatch[] = { 1, 2 };
    sregex_token_iterator pos2(
        data.cbegin(), data.cend(),    // sequence
        re,     // token separator
        arrSubMatch);
    // 0:full match, 1:First substring, 2:second substring

    for (; pos2 != end; ++pos2)
    {
        cout << pos2->str() << endl;
    }
    // 输出
    // first
    // Nico
    // last
    // Josuttis

    // -1指定显示匹配的prefix,即前面不匹配的
    string names = "nico, jim, helmut";
    regex sep("[ \t\n]\*[,;.][ \t\n]\*");  
    // separated by , ; or . and spaces
    sregex_token_iterator iter(
        names.cbegin(), names.cend(),    // sequence
        sep,    // separator
        -1);    // -1: values between separators
    for (; iter != end; ++iter)
    {
        cout << *iter << endl;
    }
    // 输出
    // nico
    // jim
    // helmut

    return 0;
}
