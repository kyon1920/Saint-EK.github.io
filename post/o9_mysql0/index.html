2019.11.14




MySQL的使用
修改密码
1>. 查看文件寻找密码
      vi /root/.mysql_secret
2>. 登录MySQL
      MySQL 服务启动和关闭
         service mysql start
         service mysql stop
      登录服务器
         mysql -u root -p密码
3>. 修改密码
      mysql> set password=password("123456");
      也可以使用脚本 /usr/bin/mysql_secure_installation 交互方式的修改mysql的root密码
4>. 退出
      quit

备份数据
mysqldump -u用户名 -p 数据库名>文件名.sql

恢复数据
mysql -u用户名 -p 数据库名<文件名.sql 或 Source 文件名.sql

和Oracle数据库的第一个区别
虚表 (以查询时间为例)
      Oracle：select sysdate from dual;
      MySQL：select now() from dual; / select now();

如果在某一天客户端登录不上服务器了，可以手动关闭守护进程，守护进程自动重启之后就可以正常登录了
命令：ps -u mysql
关闭守护进程：kill -9 mysqld的进程号

MySQL与Oracle的简介与区别
Oracle是基于用户的数据库管理系统，为一个信息系统创建用户，在这个用户之上创建表空间、表
MySQL是基于数据库的管理模型，为一个信息系统建立数据库，一个数据库基本就一个信息，在一个数据库之下有n个表
MySQL更加符合我们常规的理念

MySQL中几个常用的概念
• SQL(Structed Query Language)：结构化的查询语言
• DDL(Data Definition Language)：数据定义语言
      命令：CREATE、DROP、ALTER 等
• DML(Data Manipulation Language)：数据操纵语言
      命令：INSERT、UPDATE、DELETE
• DQL(Data Query Language)：数据查询语言
      命令：SELECT
• CRUD操作：指在做计算处理时的增加(CREATE)、查询(RETRIEVE)、更新(UPDATE)和删除(DELETE)
补：DDL是针对数据库和表结构的，其它的是针对表中的数据的

数据库的操作：创建、查看、修改、删除
表的操作：创建、查看、修改、删除、表数据的CRUD

 SQL Code 
1
2
UPDATE <表名> SET 字段1=值1 [,字段2=值2… ] [WHERE 子句 ]

      语法说明如下：
      <表名>：用于指定要更新的表名称
      SET 子句：用于指定表中要修改的列名及其列值，其中，每个指定的列值可以是表达式，也可以是该列对应的默认值，如果指定的是默认值，可用关键字 DEFAULT 表示列值
      WHERE 子句：可选项，用于限定表中要修改的行。若不指定，则修改表中所有的行
      ORDER BY 子句：可选项，用于限定表中的行被修改的次序
      LIMIT 子句：可选项，用于限定被修改的行数
• 修改数据表：

      删除字段：ALTER TABLE <表名> DROP <字段名>;
      修改字段名称：ALTER TABLE <表名> CHANGE <旧字段名> <新字段名> <新数据类型>;

MySQL中的中文乱码问题
MySQL有六处使用了字符集：system、client、server、connection、database、results
      system 是数据库系统使用的字符集设定
      client 是客户端使用的字符集
      server 是服务器安装时指定的默认字符集设定
      connection 是连接数据库的字符集设置类型，如果程序没有指明连接数据库使用的字符集类型就按照服务器默认的字符集设置
      database 是数据库服务器中某个库使用的字符集设定，如果建库时没有指明，将使用服务器安装时指定的字符集设置
      results 是数据库给客户端返回时使用的字符集设定，如果没有指明，使用服务器默认的字符集
利用 show variables like 'character%'; 可以查看以上六处使用的字符集
而我们程序需要关心的是 客户端和服务器连接 的地方
• 中文问题的三层因素：
1>. MySQL自身的设计 (6处中文)
      eg：登录时指定客户端字符集
          mysql -uroot -p --default character_set=gbk;
      它会影响数据的输入和输出
      然后使用 show variables like 'character%'; 查看字符集
      这就是影响我们程序员编程的中文问题之一，它影响了三处 (最外层)
      如果使用了 gbk 后能够插入中文呢？插入成功后能使用中文显示出来呢？
           不能，不能
2>. 操作系统的语言集设置的原因
      Linux 操作系统 是一个多用户的操作
           cat /etc/sysconfig/i18n
      操作系统的菜单按照 zh_CN显示 | 文件存储按照 utf8 显示
      Linux 操作系统的语言环境 和 用户配置的语言环境 LANG 相互影响
      修改用户的LANG (vi /.bash_profile) 去屏蔽操作系统的LANG设置也会影响数据库
      用户的LANG设置影响了应用程序的语言环境导致MySQL环境也受到了影响
      如果在这个环境之下再向数据库中插入中文数据，会成功吗？
           成功！但是查询时显示的是乱码，恐怖
      应该把自己的程序部署到单独的用户之下，这个用户的环境变量归自己/程序所有，不允许让另外任何一个程序去接触这个程序，要不然就麻烦了
      这是一个十分深刻的教训，在项目开发里面 切忌 是两个开发厂商使用同一个用户在底下部署环境
3>. 文件存储格式
