SQL监听服务 (在终端中)
⚽ 启动监听：lsnrctl start
⚽ 查询监听状态：lsnrctl status
⚽ 关闭监听：lsnrctl stop

什么是 Pro * C/C++
⚽ 通过在过程化编程语言或面向对象语言 C/C++ 中嵌入 SQL 语句而开发出的应用程序

什么是嵌入式 SQL
⚽ 在通过用编程语言中的 SQL 称为 嵌入式SQL
⚽ 在 SQL 标准中定义了多种语言的 嵌入式SQL
⚽ 各个厂商对 嵌入式SQL 的具体实现不同

什么是 Pro * C/C++ 程序
⚽ 在 C/C++ 语言中嵌入SQL语句而开发的应用程序称为 Pro * C/C++ 程序
⚽ 目的是使 C/C++ 这种高效率的语言称为访问数据库的工具

什么是宿主语言
⚽ 嵌入式SQL 的载体是宿主语言

宿主语言 Pro程序
C/C++ Pro * C/C++
FORTRAN Pro * FORTRAN
PASCAL Pro * PASCAL
COBOL Pro * COBOL
PL/1 Pro * PL/1
Ada Pro * Ada

访问数据库的方法
⚽ 用 SQL * Plus，它有 SQL 命令以交互的应用程序访问数据库
⚽ 用第四代语言应用开发工具开发的应用程序访问数据库，这些工具有 SQL * Froms、SQL * Reportwriter、SQL * Menu 等
⚽ 利用在第三代语言内嵌入的SQL语言或ORACLE库函数调用来访问

访问 Oracle数据库 的方法
✦ SQLPLUS：Oracle 自带访问数据库的工具
✦ JDBC：Java DataBase Connectivity standard
✦ ODBC：Open DataBase Connectivity 开放数据库互连
✦ OCI：Oracle Call Interface
✦ OCCI：Oracle C++ Call Interface
✦ Pro * X：Pro * C/C++、Pro * COBOL、Pro * FORTRAN、Pro * Ada

预编译工具 PROC 和编译选项
⚽ 该工具在什么地方
       cd $ORACLE_HOME，cd bin目录下
⚽ 功能：proc预编译器，把 .pc 文件 编译成为 .c 文件
       proc默认编译为c文件，而不是C++文件
⚽ 基本命令格式
       proc iname=filename [OptionName1=value1] . . . [OptionNameN=valueN]
⚽ 常用编译选项
       ﹢ INAME=path and filename (name of the input file)
       ﹢ ONAME=path and filename (name of the ouput file)
       ﹢ INCLUDE=path (头文件所在路径)
       ﹢ ...INCLUDE=路径名 或 INCLUDE=(路径名1, 路径名2)
       ﹢ PARSE=FULL | PARTIA | NONE (default FULL for C, Others for C++)
       ﹢ CODE=ANSI_C | CPP (default ANSI_C)
       ﹢ USERID=username/password
⚽ eg：proc INAME=demo_hello.pc ONAME=demo_hello.cpp PARSE=NONE CODE=CPP

#include <iostream>
#include <stdlib.h>
#include "sqlca.h"
using namespace std;

// 数据库变量定义区
EXEC SQL BEGIN DECLARE SECTION;
char* serversid = "oubc/oubc";
EXEC SQL END DECLARE SECTION;

int main()
{
    // 在C++语言中直接使用宿主变量
    cout << "serversid:" << serversid<< endl;
    // 连接Oracle，在嵌入式SQL语言中，使用宿主变量 :
    EXEC SQL connect :serversid;
    if (sqlca.sqlcode != 0)
    {
        cout << "connect SQL failed!" << endl;
        cout << "errno:" << sqlca.sqlcode << endl;
        exit(sqlca.sqlcode);
    }
    cout << "connect SQL success!" << endl;
    EXEC SQL commit release;
    return 0;
}

proc的makefile的编写
all:demo_0 demo_1
demo_0:
      @echo '========proc make demo_0========'
      proc demo_0.pc
      gcc demo_0.c -o demo_0 -I \${ORACLE_HOME}/precomp/public/ -L \${ORACLE_HOME}/lib -lclntsh

demo_1:
      @echo '========proc make demo_1========'
      proc iname=demo_1.pc parse=none code=cpp oname=demo_1.cpp
      g++ demo_1.cpp -o demo_1 -I \${ORACLE_HOME}/precomp/public/ -L \${ORACLE_HOME}/lib -lclntsh

clean:
      @rm -rf demo_0
      @rm -rf demo_1
